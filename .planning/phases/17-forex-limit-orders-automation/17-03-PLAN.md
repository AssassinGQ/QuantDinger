## Canonical `trading_config` shape (implement exactly)

```json
"trading_config": {
  "live_order": {
    "order_type": "limit",
    "max_slippage_pips": 3.0
  }
}
```

- Default: omit `live_order` or set `"order_type": "market"` → current behavior (market-only enqueue).
- **Invalid limit** (`computed limit_price <= 0`): **do not enqueue**; log and return `False` from `SignalExecutor.execute` for that signal.

## Pip size helper (concrete)

New module `backend_api_python/app/services/live_trading/forex_pip.py`:

- `def pip_size_for_forex_symbol(symbol: str) -> float:`
  - If `"JPY"` in `symbol.upper()` (covers USDJPY, EURJPY, XAGJPY-style): return `0.01`
  - Else: return `0.0001` (major/minor FX and XAUUSD-style unless JPY in name)

## Limit price formulas (from CONTEXT)

- Let `side` be `"buy"` or `"sell"` from `client.map_signal_to_side(signal_type, market_category=market_category)`.
- Let `p = float(current_price)`, `s = float(max_slippage_pips) * pip_size_for_forex_symbol(symbol)`.
- BUY: `limit_price = p + s`
- SELL: `limit_price = p - s`

## Scope

- Apply limit-path only when `market_category` in `("Forex", "Metals")` **and** `live_order.order_type == "limit"` **and** `execution_mode == "live"` path is used (same conditions as today for live IBKR enqueue via worker).
- Other categories unchanged (still market).
