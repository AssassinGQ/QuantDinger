## Locked values (17-CONTEXT)

- Limit orders for **automation** use **DAY** only; `_get_tif_for_signal` must branch on `order_type == "limit"` → `"DAY"` (ignore `market_type` for TIF when placing limits).
- **minTick snap**: BUY → `math.floor(raw / minTick) * minTick`; SELL → `math.ceil(raw / minTick) * minTick`.
- Reuse `reqContractDetailsAsync` in the same logical path as quantity alignment (extend `_align_qty_to_contract` or add `_get_contract_size_increment_and_mintick` returning `(increment, min_tick)`).

## REST `timeInForce` whitelist

- Accept optional JSON field `timeInForce` (camelCase) on `POST /api/ibkr/order` when `orderType` is `limit`.
- Allowed strings (exact, uppercased before use): `IOC`, `DAY`, `GTC`.
- Default when omitted: `DAY` (matches automation product).
- Pass through to `place_limit_order(..., time_in_force=...)` so IBKR `LimitOrder` receives that `tif`.
